To Be a Blogger

Feb 18, 2022
阅读时长: 5 分钟

🙃/To Begin

blog翻译成中文也就是博客，不知道是什么时候起，我开始发现了博客这个东西，对于刚开始了解网络的我对很多东西都有兴趣，于是我开始研究了解了一番blog的前世今生，并且也享受着别人的blog，网络世界很大，日常除了上网冲浪还是上网冲浪的我在掌握了一点点微不足道的小技巧、小知识后便产生了与人们分享的需求，刚好blog便很好满足了我的需求。

二话不说的我拿起电脑就是一顿操作，估计很多人和我一样，都是从hexo入坑的🙈，一开始进入这个“深坑”时，我感觉一切都是那么有趣新颖，什么主题和功能都想尝试一下，东搞搞西搞搞，我已经完全忘记了我搭建这个blog的目的了，一个劲的折腾各种小玩意，无法自拔，导致我也没有写下什么内容，直到我经历了多次的审美疲劳，我才慢慢的意识到我最初的目的。

是时候发挥blog应有的作用了，折腾了这么多，我明白了我真正需要的是一个稳定、方便、易于阅读和维护的blog。但这并没有结束，因为符合更多苛刻条件并且长在你审美上的theme或许不易于找到，自己写一个主题的话应该说是最有效的解决方法，但是这太麻烦，而且需要掌握某些方面的专业知识，很显然，这不适合我这种人菜瘾大的小白😩……….但很幸运，hugo和hugo-theme-stack恰好通过了我的层层筛选，拥抱它，并享受它……….

🔮/留意的问题

blog作为发布言论的地方，不是法外之地，同样也要留意一些法律问题，这涉及的很多内容，最好有个大概的了解。做到知法守法用法🎓

🍭我对blog的认识

从博客的属性和本质上讲的。blog是网络日志的意思，也就是在网上写日记。即web log。我可以用它来记录我所学的知识，用它来记录生活，独立blog便是属于你在互联网的一片自留地，我在blog里分享，并收获着喜悦。它不同于知乎、CSDN等这些大平台，你的内容属于你自己，你能够享有更大的自由，互联网时刻发生巨变，有些平台有时并不值得信任，blog才是最稳定安全的方法，源文件放在本地，由自己掌握,想随时可以发布到互联网上…….

🎈我读，我看，我写！

👻本博的出生

本博本着稳定，简单，自由，低成本诞生，GitHub、vercel和hugo是我比较信任且稳定的平台和工具，通过这些驱动我的博客是个不错的选择。为什么选择hugo而不是hexo，因为hexo需要npm，这会使得整个blog的源文件很臃肿，对网络也有更高的要求，且在文章多了之后速度会变慢很多，hugo自称是世界上最快的博客生成器也是当之无愧，所以我会更推荐使用hugo。

搭建本博客其实很简单，我也不是很喜欢重复造轮子，这里给出搭建blog的相关文档🧀

🪅我的博客魔改

🎈魔改声明

魔改的内容采用多个博客站点的样式并进行删减整合，如侵权请留言删除

由于hugo-theme-stack在某些方面我认为是不易于阅读和使用的，因此我对其进行了小小的改造。对hugo主题进行改造不应该直接修改theme目录下的主题源码，这里我将给出我的整个blog文件夹目录结构并作出简单的解释。这里引用文档对修改主题的介绍

修改主题方法:

主题预留了 layouts/partials/head/custom.html 和 layouts/partials/footer/custom.html 这两个文件来自定义页面头部和底部的内容。

借助 Hugo 的特性可以很方便地覆盖主题的源码（类似 WordPress 的子主题功能）。

举个例子，假设我要在页面底部添加 Google Analytics 统计代码，可以这么做：

  1. 在站点根目录新建 layouts/partials/footer/custom.html 文件
  2. 在该文件内插入统计代码

这么做的好处就是可以直接覆盖升级主题，同时不丢失你对主题做的修改。

也可以用同样的方式修改主题的 SCSS 文件（存放在 assets/scss 文件夹下）。同理，需要在站点根目录下创建同名文件，Hugo 就会赋予它更高的优先级。

我的blog文件夹目录：

blog:.
├─archetypes #生成带有YAML Front Matter模板的文章
├─assets
│  ├─icons #存放魔改的图标的文件夹
│  ├─img #存放魔改的图片的文件
│  └─scss #存放魔改的scss样式的文件夹
├─content #博客的主要内容
│  ├─categories #用来放分类
│  ├─page #用于存放我的页面，如
│  │  ├─about #关于
│  │  ├─archives #归档
│  │  ├─links #友链
│  │  ├─louxiang #陋巷
│  │  └─search #搜索
│  └─post #用来放文章
├─layouts #用于魔改主题里的layouts文件
├─public #最终生成blog的目录
├─resources #无需了解
├─static #存放静态文件的目录，如CNAME等
│  └─img
└─themes #存放主题的文件夹
    └─hugo-theme-stack

友链页需要使用短代码即shortcode实现，于是需要在blog根目录新建layouts\shortcodes\friend.html

{{- $name := .Get "name" | default (.Get 0) -}} {{- $url := .Get "url" | default (.Get 1) -}} {{- $avatar := .Get "avatar" | default (.Get 2) -}} {{- $bio := .Get "bio" | default (.Get 3) -}}

<a href="{{- $url -}}" title="{{- $name -}}" class="friend-link" target="_blank" rel="friend">
    <div class="friend-link-div">
        <div class="friend-link-avatar">
            <img src={{ .Get "avatar" }} class="friend-avatar" loading="lazy" alt="Avatar">
        </div>
        <div class="friend-link-info">
            <i class="fa fa-link" aria-hidden="true"></i>
            <i class="friend-name">{{ $name }}</i>
            <p class="friend-bio">{{ $bio }}</p>
        </div>
    </div>
</a>
